SECTION A — seven business types (classify into exactly one; PRIOR for where to search, not a source of answers)

- infra_usage_based: revenue scales with API calls / volume consumed. Wide open, key-based docs expected.
- saas_seat_based: per-user subscription. Open API as retention.
- ad_platform: revenue from ads / attention on their surface. Docs public; production often needs app review.
- data_vendor: proprietary dataset is the product. API often gated/expensive.
- commerce_platform: merchants pay to sell through them. Per-merchant API; often needs shop URL.
- enterprise_sales: contract sales only, no self-serve. Docs on request; partnership.
- ai_native: recently launched AI product. Inconsistent posture.

Do NOT use open_source as a business_type. Open-source status is a separate fact (is_open_source) filled in Call 2.
If a product has cloud + self-host, search the CLOUD path.

State business_type and a one-sentence business_type_reason.
