Emit these fields:
- one_liner (max 15 words)

- docs_access: public | login_required | on_request | none_found
- docs_location: own_domain | docs_subdomain | third_party_host | none

- auth_secondary: array of the same auth enum (may be empty)
- auth_detail: one line free text (header names, query vs header, key+secret, subdomain as credential)
- access_tier: open | self_serve_free | self_serve_trial | card_required | plan_gated | approval_gated | partner_gated | no_public_access
  (self_hosted is NOT allowed)

- api_type: rest | graphql | both | soap | sdk_only | cli_tool | none | unknown
- api_breadth: narrow | medium | broad | unknown
- has_openapi_spec: yes | no | unknown
- needs_instance_url: yes | no | unknown
- has_webhooks: yes | no | unknown
- rate_limit_note: free text
- mcp_exists: official | community | none | unknown
- mcp_access: same_as_api | paid_only | waitlist | n_a
- evidence: object mapping field_name -> URL (must be a SOURCE URL from the input)
- confidence: object mapping field_name -> high | med | low
- notes: free text

CLOUD PATH ONLY: if the product has cloud and self-host options, classify the cloud/hosted path. Mention self-host only in notes.

CLI tools (local commands, no vendor API): api_type=cli_tool.

SECTION B — evidence rule
Every non-unknown field must cite a source URL that appears in the provided input under === SOURCE: url ===. If the pages do not state something, answer unknown. Do not use training knowledge. Do not infer.

SECTION C — tag stability
- Extra procedural steps do not change a tag.
- Header vs query-param, key+secret, multi-step flows belong in auth_detail, not auth_primary.
- When both API key and OAuth exist: docs-lead for auth_primary; the other in auth_secondary.
- other is never valid. Use unknown + notes.

SECTION D — output format
Strict JSON matching the fields above. No markdown fences. No verdict fields.
