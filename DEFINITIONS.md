# Definitions

Closed enums and stated assumptions for the Composio Toolkit Research Agent.
Mirrors `schema.py`. Superseded where `LOGIC_FREEZE.md` conflicts with `SPEC.md`.

---

## Stated assumption: public / distributed app path

**Composio ships toolkits to many customers.** Verdicts assume the **public / distributed** integration path, not a single-tenant custom or private app.

Consequences:
- Apps that require vendor app review, partner program approval, or production access review before a multi-customer connector can ship are tagged `needs_review` (or worse), even when a custom/private app key would be an easy win for one engineer.
- This applies especially to Shopify, Slack, Meta Ads, and similar platforms.

This assumption is frozen before ground truth is filled and must appear on the findings site.

---

## Stated finding slot: CLI tools are not connector candidates

Entries with `api_type=cli_tool` (e.g. Mermaid CLI, Sherlock) are local commands with no vendor-authenticated API surface. They are **not** Composio connector candidates. Site findings must surface this as a category note.

---

## Stated note: `wait_class` is coarse

`wait_class` is a **fixed map** from `buildability`, not an estimated calendar duration:

| buildability   | wait_class |
|----------------|------------|
| easy_win       | none       |
| easy_but_paid  | none       |
| needs_review   | weeks      |
| needs_outreach | months     |
| blocked        | n_a        |
| unknown        | unknown    |

## Business classification

**`business_type`** — set by Call 1 as a prior. Call 2 must **not** overwrite it; Call 2 sets `business_type_confirmed` only (`yes` / `no`).

| tag | definition | expected API posture |
|---|---|---|
| `infra_usage_based` | revenue scales with API calls, compute, or volume | wide open, key-based |
| `saas_seat_based` | per-user subscription | open API as retention |
| `ad_platform` | revenue from ads / attention on their surface | docs public; production often needs app review |
| `data_vendor` | proprietary dataset is the product | API gated or expensive |
| `commerce_platform` | merchants pay to sell through them | open per-merchant; often needs shop URL |
| `enterprise_sales` | contracts; no self-serve | docs on request; partnership |
| `open_source` | public source; may also have a cloud product — **classify the cloud path** | auth configured by host for self-host; cloud path may still gate |
| `ai_native` | recently launched AI product | inconsistent |

Self-host options belong in `notes` only, never in scored fields.

## Documentation

**`docs_access`:** `public` · `login_required` · `on_request` · `none_found`

**`docs_location`:** `own_domain` · `docs_subdomain` · `third_party_host` · `none`

Vendor-controlled pages on GitHub, Stoplight, Readme.io, Mintlify, GitBook, Postman, Apiary, Swagger Hub count as first-party evidence. Blogs and aggregators never do.

## Authentication

**`auth_primary` / `auth_secondary`:** `api_key` · `pat` · `oauth2` · `oauth1` · `basic` · `jwt_keypair` · `hmac_signed` · `aws_sigv4` · `session_only` · `none` · `unknown`

When both API key and OAuth exist: **docs-lead** for primary; the other in secondary. Mechanics in `auth_detail` (never clustered).

`session_only` → buildability `blocked`.

## Access

**`access_tier`:** `open` · `self_serve_free` · `self_serve_trial` · `card_required` · `plan_gated` · `approval_gated` · `partner_gated` · `no_public_access`

`self_hosted` is **retired**. Cloud path only.

## API surface

**`api_type`:** `rest` · `graphql` · `both` · `soap` · `sdk_only` · `cli_tool` · `none` · `unknown`

**`api_breadth`:** `narrow` · `medium` · `broad` · `unknown`

**`has_openapi_spec` / `needs_instance_url` / `has_webhooks`:** `yes` · `no` · `unknown`

## MCP

**`mcp_exists`:** `official` · `community` · `none` · `unknown`

**`mcp_access`:** `same_as_api` · `paid_only` · `waitlist` · `n_a`

## Buildability (derived in code — never written by Gemini)

| tag | meaning |
|---|---|
| `easy_win` | credentials today, no spend, no external human |
| `easy_but_paid` | mechanically simple; internal spend approval only |
| `needs_review` | vendor must approve before production |
| `needs_outreach` | no self-serve; BD / partner path |
| `blocked` | no usable API / CLI-only / session-only / no docs |
| `unknown` | insufficient evidence |

**`blocker_type`:** `none` · `payment` · `vendor_review` · `partnership` · `no_api` · `no_docs` · `unknown`

**`unblocker`:** `nobody` · `composio_finance` · `vendor_human` · `composio_bd` · `n_a`

**`blocker`:** one-line string generated by `derive_verdict()`; empty when `easy_win`.

## Design rules

- Tag columns are closed enums. No free values. No `other` — use `unknown` + `notes`.
- Free-text columns are for human debugging only; never clustered.
- Extra procedural steps never change a tag.
- Unknown remains a valid terminal answer.
- Verdict fields are derived from atomic facts by a pure function shared with ground truth.
