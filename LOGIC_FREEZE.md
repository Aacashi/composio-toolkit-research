## 1. Second round triggers

Trigger a targeted second round when ANY of these is unknown:
`access_tier`, `auth_primary`, `api_type`, `has_openapi_spec`, `has_webhooks`, `mcp_exists`.

Priority for second-round search/fetch budget:
`access_tier` > `auth_primary` > `api_type` > openapi > webhooks > mcp.

`access_tier` and `auth_primary` are the scored fields — they get pages first.
Still max ONE targeted round per app. Cap second-round search tool calls at 2.

## 2. Merge rule

Second round fills unknowns ONLY. Never overwrite a field that already has a
value and a valid evidence URL. Contradictions go in `notes`. Same first-party
+ guard rules. No exceptions.

## 3. Fixed `wait_class` map

| buildability | wait_class |
|---|---|
| easy_win | none |
| easy_but_paid | none |
| needs_review | weeks |
| needs_outreach | months |
| blocked | n_a |

State on the site that `wait_class` is a coarse fixed class, not an estimate.

## 4. Verdict priority (worst wins; first match top-down)

1. `api_type` in {cli_tool, none} OR `docs_access=none_found` → blocked / n_a / n_a / no_api (or no_docs)
2. `auth_primary=session_only` → blocked / n_a / n_a / no_api
3. `access_tier` in {partner_gated, no_public_access} → needs_outreach / composio_bd / months / partnership
4. `access_tier=approval_gated` → needs_review / vendor_human / weeks / vendor_review
5. `access_tier` in {card_required, plan_gated} → easy_but_paid / composio_finance / none / payment
6. `access_tier` in {open, self_serve_free, self_serve_trial} → easy_win / nobody / none / none
7. else → unknown / n_a / unknown / unknown

`self_hosted` retired from `access_tier`. Cloud path only.

## 5. CLI short-circuit

`api_type=cli_tool` forces atoms + verdict before/as derivation:
`docs_access=public`, `docs_location=third_party_host`, `access_tier=no_public_access`,
`auth_primary=none`, buildability blocked / no_api. Guard skips evidence for short-circuit fields.

## 6. `blocker` string

Generated by `derive_verdict()`. Gemini never writes it. Empty when `easy_win`.
